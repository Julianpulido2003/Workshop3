Un administrador de red digita el comando de red mostrado en laFigura 3. Explique el significado de: (1) bytes=32, (2)tiempo=7ms, y (3)TTL=64.

Los 32 Bytes se refieren al tamaño del paquete de datos enviado para comprobar conectividad con la página o con la dirección solicitada. El tamaño de cada paquete puede cambiar dependiendo el tipo de conexión o la especificación que el usuario requiera, pues también se pueden mandar de 64 Bytes entre otros valores. El tiempo nos muestra cuanto tiempo se ha tardado aproximadamente en establecer conexión mandando los cuatro paquetes, en el caso del ejemplo podemos ver que el promedio está entre 3ms y 7ms. Finalmente, TTL=64 o Time To Live es una manera de evitar que una petición de ping viaje sin fin por el intenet. Se refiere a la cantidad de “saltos” que da el paquete de host en host hasta alcanzar su destino, por cada salto el valor disminuye, cuando se llega a 0 el paquete es descartado y se arroja un mensaje de error, pues significa que fue imposible establecer conexión.

Un administrador de red digita el comando de red mostrado en la Figura 5. Explique los términos Destino de red, Máscara de red, Puerta de acceso, Interfaz, Métrica

Lo primero que hay que tener en cuenta es que el comando netstat -r funciona para darnos las tablas de enrutamiento de nuestro dispositivo, estas tablas son usadas por el sistema operativo para poder comunicarse con otros dispositivos o redes. Estas tablas muestran los componentes: Destino de Red: Se refiere a la dirección IP a la que se quiere llegar o que se quiere alcanzar. Máscara de Red: Es importante, pues gracias a esta, se puede saber en qué red se está trabajando. Puerta de acceso: Establece por que puerta de enlace es necesario pasar para llegar al Destino de Red. En ciertos casos, puede salir que está en vínculo, esto se refiere a que no se necesita una puerta de enlace para alcanzar el destino. Interfaz: Establece que interfaz es necesaria usar para poder llegar al Destino de Red. Métrica: Es un valor que se le asigna a cada IP para una interfaz determinada. Se puede decir que es el costo de la ruta, se le puede dar valor por velocidad de vínculo o recuento de saltos necesarios para llegar al Destino de Red.

Un administrador de red digita el comando de red mostrado en la Figura 7. ¿Qué significa Traza? ¿Qué indica el número de saltos? ¿Qué significa el <1 ms en cada salto? ¿Para qué se realiza una Traza?¿En qué momento es útil utilizar el comando Tracert?

Una traza se refiere a una captura del tráfico de red que se envía entre dos o más dispositivos para de esta manera comprobar y estudiar la forma en la que están trabajando dichos dispositivos, se puede decir que es el recorrido que se hace para enviar datos. El número de saltos, de manera similar a el comando “ping” determina los enrutadores intermedios que tuvo que pasar la información hasta su destino. El tiempo en los saltos se refiere la cantidad aproximada de tiempo que se empleo para poder completar cada salto Las trazas tienen dos funciones principales, la primera es para estudio, pues al capturar todo el tráfico de red de ciertos dispositivos, se pueden comprobar si hay alguna falencia en el sistema de comunicación. Por otro lado, su otra funcionalidad es seguridad, pues de esta manera se puede comprobar toda la información que entra y sale de un dispositivo y de esta manera verificar que no haya información en lugares equivocados. El comando Tracert principalmente se usa cuando hay alguna dificultad para establecer conexión entre dos puntos, pues si bien el comando Ping nos manifiesta que no se puede conectar, el Tracer nos desglosa el camino paso a paso de la información y nos ayuda a desvelar de manera más rápida donde está el problema.
